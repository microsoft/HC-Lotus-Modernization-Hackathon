Risk Position

This repository is designed for safe collaboration across organizations. Controls focus on preventing accidental exposure while keeping delivery velocity high.

Enforcement Model

  • Redaction and dummy-data expectations are explicit
  • Commit-time review checklist is documented
  • Prohibited content categories are clearly defined
NO

Never Commit Sensitive Data

No real user data, no production exports, no credentials, and no internal HC infrastructure details.

ONLY

Use Approved Inputs

Anonymized screenshots, schema metadata, and clearly fake sample data are allowed for build and testing.

CHECK

Review Before Every Commit

Redact screenshots, remove endpoints, and verify no secrets or PII are present in staged files.

Allowed Content

  • Anonymized screenshots
  • Field names, types, relationships
  • Dummy or randomized datasets
  • Application code, tests, and architecture docs

Prohibited Content

  • Production database exports
  • Real names, emails, identifiers
  • Internal HC URLs or IPs
  • API keys, passwords, access tokens

Before Every Commit

  • Redact screenshots for names, emails, case IDs, and any PII
  • Confirm no production records, secrets, or internal endpoints
  • Use only dummy datasets and anonymized schema references

If Sensitive Data Is Found

  • Notify the team immediately and stop distribution
  • Purge from git history, not just latest commit
  • Rotate exposed credentials and re-verify repository state
