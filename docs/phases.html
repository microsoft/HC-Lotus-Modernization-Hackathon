01

Discover And Frame

Capture legacy context: workflows, schemas, constraints, and operating assumptions.

AI Does

  • Structure and summarize captured artifacts
  • Draft workflow narratives and data model descriptions
  • Identify gaps and request clarification

DTB Team Does

  • Provide source context and business nuances
  • Validate and correct generated documentation
  • Approve baseline before moving to Phase 2

Outputs

  • Validated context package
  • Workflow inventory with decision points
  • Baseline constraints and data boundaries
Prompts, Examples & Time Allocation

Example Prompts

  • "Based on these screenshots, document this application's UI, forms, and navigation flow."
  • "Given this data schema, describe the data model, relationships, and likely validation rules."
  • "Document the business rules and workflows visible in these screens."

Time Allocation

Typically 60–90 minutes. Front-loaded with artifact collection, then AI synthesis, then team review.

Quality Gate

DTB team confirms: "This documentation accurately represents our current system." No code begins until this gate passes.

02

Define Requirements

Translate discovery into implementation-ready requirements, user stories, and acceptance criteria.

AI Does

  • Generate draft functional requirements from documentation
  • Propose user stories in standard format
  • Suggest non-functional criteria and acceptance checks

DTB Team Does

  • Review, edit, and prioritize generated requirements
  • Confirm alignment to operational realities
  • Define what is in-scope vs. future work

Outputs

  • Prioritized requirements document
  • User stories with acceptance criteria
  • Scope boundaries (in/out for this engagement)
Prompts, Examples & Time Allocation

Example Prompts

  • "Based on this documentation, generate user stories in the format: As a [role], I want to [action], so that [benefit]."
  • "What are the key functional requirements to replicate this application as a modern web app?"
  • "Suggest non-functional requirements for performance, accessibility, and security."

Time Allocation

30–45 minutes. AI generates quickly; team time is spent on prioritization and scope decisions.

Quality Gate

DTB team confirms: "These requirements represent what we need to build." Architecture work begins only after this approval.

03

Plan Conversion

Define architecture, migration sequencing, and implementation boundaries before coding begins.

AI Does

  • Propose target architecture options with tradeoffs
  • Generate migration sequencing and phased delivery plan
  • Draft database schema and API boundaries

DTB Team Does

  • Select architecture approach and finalize decisions
  • Approve phased rollout and risk controls
  • Generate test data matching approved schema

Outputs

  • Target architecture document
  • Ordered conversion plan (build sequence)
  • Dummy/test data for development
Prompts, Examples & Time Allocation

Example Prompts

  • "Propose a modern application architecture for these requirements. Include database schema, API design, and component breakdown."
  • "Generate a SQL schema based on this field definition."
  • "Create a seed data script with 50 randomized records matching this schema."

Time Allocation

30–45 minutes. Architecture decisions are the bottleneck — AI proposes fast, team deliberates.

Quality Gate

DTB team confirms: "This architecture and plan are approved for implementation." Coding begins only after this sign-off.

04

Execute And Deliver

Build modernization increments with AI-assisted coding and mandatory human review at every merge.

AI Does

  • Generate code, tests, and infrastructure from the plan
  • Assist with debugging and iteration acceleration
  • Suggest refactoring and documentation improvements

DTB Team Does

  • Validate correctness, security, and maintainability
  • Own final code quality and production readiness
  • Document decisions and capture reusable patterns

Outputs

  • Working application increment
  • Automated tests and validation suite
  • Reusable prompt patterns and delivery playbook
Prompts, Examples & Time Allocation

Example Prompts

  • "Generate an Entity Framework model for this database schema."
  • "Create a REST API controller for CRUD operations on [entity]."
  • "Build a React component that displays this data in a table with filtering and sorting."
  • "Write unit tests for this business logic."

Time Allocation

3+ hours (bulk of engagement). Rapid iteration cycles: AI generates → human reviews → AI refines.

Quality Gate

DTB team confirms: "This increment works, is tested, and meets our standards." Demo-ready output before wrap-up.
